2  Random Intercepts

2.1 Why Linear Mixed-Effects Models?

In the section recapping linear models, we used a linear model to describe the relationship between word frequency and log response time in a lexical decision task. In these data we had one observation per word.

In a real study, however, we would have multiple observations for each word from each participant. The traditional approach would be to aggregate: calculate the average log response time for each word before fitting the model. However, while this allows us to account for between-item variability, by aggregating over participants, we are losing valuable information about how the data were generated.

Where it was possible, a common compromise could be to fit the model twice, and then examine whether a hypothesis is supported across both items and participants (Raaijmakers et al. 1999). However, a more robust solution would be to model item and participant variability simultaneously. This is something that linear-mixed effects models allow us to do (Locker et al. 2007).

2.2 What is a Linear Mixed-Effects Model?

A linear mixed-effects model is largely a generalisation of the ideas behind a linear model that we went over in the recap on linear models. A key difference is that rather than aggregating, we can model multiple sources of variability simultaneously.

To start with, we will just work with random intercepts:

\[logRT_{ip} = \beta_0 + I_{0i} + P_{0p} + \beta_1 Zipf_{ip} + e_{ip}\] Our formula is very similar to before, with the following key differences:

  • We index observations with two indices: \(i\) (item) and \(p\) (participant).
  • We include \(I_{0i}\): per-item random intercepts
  • We include \(P_{0p}\): per-participant random intercepts

As we saw with \(e_i\) in the linear model example, \(I_{0i}\), \(P_{0p}\), and \(e_{ip}\) are all drawn from normal distributions with their own standard deviations:

\[\begin{aligned} I_{0i} &\sim \mathcal{N}(0, \sigma_{I0}) \\ P_{0p} &\sim \mathcal{N}(0, \sigma_{P0}) \\ e_{ip} &\sim \mathcal{N}(0, \sigma_e) \end{aligned}\]

The per-item and per-participant random intercepts describe variability among these populations. Items will vary in their average response time — while word frequency is maybe the best predictor of response times, a whole range of other factors will also affect a single word’s average response time. Similarly, participants will differ in their average response times, with some participants responding consistently faster, and some responding consistently slower. Our model says that these random effects combine with our fixed effects to generate our observations.

To get a sense for what this looks like, in this section we’ll use these formulae to simulate some data.

2.3 Packages

We’ll use the following packages in this section:

library(lme4)  # for fitting LMMs
library(dplyr)  # for data wrangling
library(tidyr)  # for data wrangling
library(readr)  # for importing data
library(ggplot2)  # for plotting

2.4 Defining Parameters

Here we define all the free parameters from the equations above:

  • \(\beta_0\) (Fixed-effect Intercept): b_0
  • \(\beta_1\) (Fixed-effect Slope): b_1
  • \(\sigma_{I0}\) (SD of per-item random intercepts): s_P0
  • \(\sigma_{P0}\) (SD of per-participant random intercepts): s_I0
  • \(\sigma_e\) (SD of the residual): s_e

The numbers here are based on the same real data as in the section on linear models.

# Fixed-effect intercept and slope
# (same as the parameters in the section on linear models)
b_0 <- 6.85
b_1 <- -0.1

# SD of per-item random intercepts
s_I0 <- 0.08

# SD of per-participant random intercepts
s_P0 <- 0.16

# residual SD
s_e <- 0.33

2.5 Simulating Items and Participants

To be able to simulate trials (a single participant responds to a single item), we first need to simulate the individual items and participants. Here we build two dataframes, of 50 items and 30 participants.

The simulated random intercepts, I_0i and P_0p, quantify how each item and participant deviate from the fixed-effect average effect.

# simulate 50 items
n_items <- 50

items <- tibble(
  # item IDs
  i_id = 1:n_items,
  # item word frequencies
  zipf = runif(n_items, 1.5, 7.5),
  # item intercepts
  I_0i = rnorm(n_items, 0, s_I0)
)

# simulate 30 participants
n_participants <- 30

participants <- tibble(
  # participant IDs
  p_id = 1:n_participants,
  # participant intercepts
  P_0p = rnorm(n_participants, 0, s_P0)
)

2.6 Visualising Random Intercepts

As you can see, the random intercepts allow us to describe multiple sources of variability at the same time. A common way of understanding random effects is to plot separate linear relationships for each unit in our population.

For example, just as we can plot a single simulated linear relationship, so too can we plot how this would look for items and participants in a given sample:

# (a)
ggplot(items) +
  geom_abline(aes(intercept = b_0+I_0i, slope=b_1, group=i_id)) +
  geom_abline(intercept=b_0, slope=b_1, colour="red") +
  lims(x=c(1.5, 7.5), y=c(5, 7.5))
# (b)
ggplot(participants) +
  geom_abline(aes(intercept = b_0+P_0p, slope=b_1, group=p_id)) +
  geom_abline(intercept=b_0, slope=b_1, colour="red") +
  lims(x=c(1.5, 7.5), y=c(5, 7.5))

Here, the red line shows the fixed effect, while the black lines show variability between (a) items and (b) participants. You can clearly see that there is less variability between items (s_I0 = 0.08) than between participants (s_P0 = 0.16).

Note

Panel (b) shows each participant’s predicted average response time to a stimulus of any given frequency. However, we can’t interpret panel (a) in this way, because each item only has one zipf value! Here is a version of the plot showing where the simulated items actually fall on this relationship:

ggplot(items) +
  geom_abline(aes(intercept = b_0+I_0i, slope=b_1, group=i_id),
              colour="grey") +
  geom_point(aes(x = zipf, y = I_0i + b_0 + b_1*zipf),
             shape=3) +
  geom_abline(intercept=b_0, slope=b_1, colour="red") +
  lims(x=c(1.5, 7.5), y=c(5, 7.5))

Rather than the black lines in panel (a) showing the effect of frequency for each item, it shows what the predicted response time would be if it had a different frequency.

2.7 Simulating Trials

In a trial, a single participant responds to a single item. We simulate trials by simulating the meeting between participants and items.

As DeBruine and Barr (2021) do, we’ll simulate this using the crossing() function from tidyr. This simulates a study in which all 30 participants respond to 50 items, producing a total 1500 trials. For each unique combination of item and participant, we also simulate the residuals, \(e_{ip}\).

# generate trials
trials <- crossing(items, participants) |>
  mutate(e_ip = rnorm(n(), 0, s_e))

# print the first few rows
print(trials)

Note

As well as simulating all possible combinations of items and participants, you could also simulate other combinations of participants and items. For example, you could simulate participants responding to N random trials, or you could create 10 separate stimulus lists, with 10% of your participants being assigned to each set of items. Similarly, you could use real data to inform your simulation, such as taking real items and real Zipf values from a frequency corpus.

2.8 Simulating Responses

Now that we have simulated trials, we can use our formula to simulate each participant’s response to each item.

As a quick reminder, here is the formula again:

\[logRT_{ip} = \beta_0 + I_{0i} + P_{0p} + \beta_1 Zipf_{ip} + e_{ip}\]

We use this formula to add the simulated log response times to the trials dataframe:

trials <- mutate(
    trials,
    log_rt = b_0 + I_0i + P_0p + b_1 * zipf + e_ip
  )

If we plot these observations coloured by items or participants, you can see how item and participant variability is reflected in the structure of the data:

# (a)
ggplot(trials, aes(zipf, log_rt, colour=factor(i_id))) +
  geom_point() +
  theme(legend.position="none")
# (b)
ggplot(trials, aes(zipf, log_rt, colour=factor(p_id))) +
  geom_point() +
  theme(legend.position="none")

2.9 Parameter Retrieval

We can now fit a model that estimates the parameters we originally used to simulate the data. If you compare these estimates to the values we originally simulated, you can see that the estimates from our simulated sample are close to the ground truth.

m <- lmer(
  log_rt ~ 1 + zipf +
    (1 | i_id) +
    (1 | p_id),
  data = trials
)

summary(m)

2.10 lme4 Syntax

Here is a quick breakdown of the lme4 syntax in the lmer() model:

  • log_rt ~ 1 + zipf — as in the linear model formula, this says that we want to model log_rt as a function of a fixed-effect intercept (1) and a slope that should be multiplied by zipf.

  • (1 | i_id) — this says that we want to model random intercepts for each unique value of i_id (the item IDs).

  • (1 | p_id) — this says that we want to model random intercepts for each unique value of p_id (the participant IDs).

2.11 Visualising Estimated Random Effects

Just as we visualised the ground-truth relationships between zipf and log_rt (see here), we can visualise the relationships estimated by our model.

To do this, we want to extract the estimated fixed effect and the estimated per-participant random intercepts from our model.

The fixef() function gives us a vector of all fixed effects in the model.

# extract fixed-effect estimates
fes <- fixef(m)

print(fes)
(Intercept)        zipf 
 6.79232805 -0.07539171

The ranef() function gives us the random-effect estimates for all participants and items. Here, we extract the per-participant estimates, and coerce them to a dataframe.

# extract random-effect estimates
res <- ranef(m)

# extract participant random effects specifically
res_p_df <- res$p_id |>
  # coerce to a dataframe
  as_tibble(rownames="p_id") |>
  # rename the "(Intercept)" column to remove the parentheses
  rename(est_P_0p = `(Intercept)`)

print(res_p_df)

Recall that the random effects are per-unit (e.g., per-participant) deviations from the fixed effects. We can add the random intercept to the fixed-effect intercept to get per-participant predictions of average response times across Zipf.

ggplot(res_p_df) +
  geom_abline(aes(intercept = fes[["(Intercept)"]]+est_P_0p,
                  slope=fes[["zipf"]],
                  group=p_id)) +
  geom_abline(intercept=fes[["(Intercept)"]],
              slope=fes[["zipf"]],
              colour="red") +
  lims(x=c(1.5, 7.5), y=c(5, 7.5))

2.12 Cheating, with predict()

Instead of calculating the predictions manually, there is also a handy predict.merMod() method which we can use on lmer() models. The advantage of this is that you don’t have to think about the mixed-effects model formula — you can just get this information from the lme4 model.

Here, we create a dataframe called participant_preds that contains all participant IDs. These are repeated for values of Zipf from 1.5 to 7.5.

participant_preds <- expand_grid(
  p_id = unique(participants$p_id),
  zipf = seq(1.5, 7.5, length.out=1000)
)

We can then pass this dataframe to the predict() function, to predict the relationship between Zipf and log response time for each participant in our sample.

Importantly, we also have to change the re.form argument from the default. By default, the function will try to use the full random-effect formula. However, we only want to get average predictions for participants, rather than predictions of participant responses to specific items.

participant_preds$pred_log_rt <- predict(
  # pass the model
  m,
  # pass the data we want to get predictions for
  newdata = participant_preds,
  # only include the participant random intercepts
  # in the random effect formula
  re.form = ~(1 | p_id)
)

Finally, we can plot the predicted relationship between zipf and log_rt for each of our participants:

ggplot(participant_preds, aes(zipf, pred_log_rt, group=p_id)) +
  geom_line() +
  geom_abline(intercept=fes[["(Intercept)"]],
              slope=fes[["zipf"]],
              colour="red") +
  lims(x=c(1.5, 7.5), y=c(5, 7.5))
